FI-004 — CSV Batch Invoice Import
Parse multi-invoice supplier-portal CSV exports directly into the bill queue. One file → many bills, instant, no Claude. Formats covered: Selected-Invoices and SMARTBIZ.
🟢 Shipped v1.4.0 Frontend-only papaparse Zero API cost Duplicate detection
Overview Formats Detection UX Flow Queue Lifecycle API & Data Edge Cases Test Results Build Phases
Section 01
Why CSV matters
Most large suppliers (Reece, Plumbers Co-op, Bunnings Trade, Tradelink) offer a CSV export of recent invoices from their customer portal — typically a single file with 10-100 invoices covering a week or a month. Running each invoice through Claude Vision would cost $2-20 per file and take several minutes; CSV is already structured, so we parse it locally in milliseconds, group line items into individual bills, and drop them straight into the same processing queue that PDF imports populate.
Zero API cost · Zero round-trip latency. 16 invoices with 32 line items parse in under 30 ms on a mid-range laptop. No Claude call, no server hit, no Heroku egress.

Section 03
Format detection (3-layer sniff)
The uploader sniffs the file to decide which parser to invoke. Detection happens client-side, instantly, before any UI is shown.
1
File extension + MIME check Frontend
File ends with .csv or MIME is text/csv → treat as CSV (not PDF). Dropped alongside PDFs — the uploader separates streams.
2
First-row signature match Frontend
Read the first line of the file text. If it matches ^"?Document Type"?,\s*"?Customer Number"? → Format A (header-based). If the first cell is literally INV or CRD followed by ≥19 fields → Format B (SMARTBIZ positional). Anything else → unknown, reject with friendly error.
3
Parse + group Data
Route to the matching parser. Both parsers read all rows and group line items by invoiceNo into a Map. Credit notes (docType === 'CRD') have their subtotal/GST/total sign-flipped to negative so the queue totals add up correctly.
Section 04
UX flow — drop → preview → commit
A focused 3-step flow, no wizard, no multi-screen state machine. The user sees a preview and confirms a supplier; everything else is automatic.
A
Drop CSV onto uploader
Same drop zone as PDFs — it detects the extension and routes appropriately. No separate "CSV upload" entry point. Multiple CSVs → one modal per file (serial, not concurrent, to avoid modal stacking).
B
Detection banner + supplier autocomplete
Modal opens showing format detected, invoice count, total $ value, date range, and (for SMARTBIZ) the supplier ABN. Supplier autocomplete pre-fills when ABN matches an existing supplier or a remembered mapping; otherwise the user picks one. A preview table shows the first 5 invoices.
C
Import — one click, N bills
Clicks "Import 16 invoices as bills" → all 16 land in the queue, each pre-matched to a PO using the same tiered matcher PDFs use (customerPoRef === PO.id → reference → jobRef → supplier-scoped fallback). Duplicates (same supplierInvoiceNo already in queue) auto-skip with a note. ABN → supplier mapping is remembered for next time.
Section 05
Queue lifecycle — 9 states
CSV-imported bills use the exact same queue lifecycle as PDF-imported bills. The queue is persisted to localStorage (bill-importer-queue-v1), so it survives page refresh and browser restart until explicitly cleared.
Pending
Just added — not used by CSV import (CSVs are parsed synchronously). PDF-only.
Processing
Claude call in flight. PDF-only.
Auto-Matched
Linked to a PO, variance ≤ 2%. Ready to one-click apply.
Needs Review
Linked to a PO but variance > 2%. Awaits human sanity check.
No PO Found
No match. Inline Autocomplete in the row lets the user link a PO manually.
WIP
User opened for reconciliation — editing in progress.
Completed
Bill created, linked to PO, synced. Terminal state (but reopenable).
Skipped
Parked — duplicate, credit-note deferral, or user choice.
Error
Parse failure. PDF-only (CSVs either parse or reject at upload).
Filter tabs at the top: All · To Do · WIP · Completed · Skipped — badge counts update live. To-Do collapses all active statuses (pending, processing, auto-matched, needs-review, no-po, error) into a single queue of work.

Section 07
Edge cases & how we handle them
Edge caseBehaviour
Credit note (CRD) Subtotal / GST / total are sign-flipped to negative. Row amount rendered red. Queue still totals correctly.
Duplicate invoice # If supplierInvoiceNo already exists in the queue, new row lands as Skipped with note "Duplicate invoice # already in queue". User can Reopen if intentional.
Empty customerPoRef Row lands as No PO Found — inline Autocomplete in the row lets user link a PO by id / reference / job # without leaving the queue.
Unknown supplier ABN SMARTBIZ: ABN present but no supplier record matches. User manually selects supplier in the preview modal; the mapping is then remembered for next time.
Mixed dates (DD/MM/YYYY vs YYYY-MM-DD) Normaliser tries three formats: DD/MM/YYYY, YYYY/MM/DD, YYYY-MM-DD. Anything else is passed through as-is rather than rejected.
Leading-zero invoice #s Read as strings (not numbers) throughout — no padding/stripping.
Trailing blank lines papaparse skipEmptyLines:true handles. Rows missing an invoice # are silently ignored.
Multiple CSVs dropped at once Currently modal-serialised (one at a time). Queued CSVs wait for the previous modal to close — prevents modal stacking and supplier confusion.
Unknown CSV format Friendly alert(): "Could not detect a known CSV format. Expected Selected-Invoices or SMARTBIZ format." No queue pollution.
Supplier-ABN drift (format changes) If the remembered ABN points to a supplier that's since been deleted, the lookup falls back to DB ABN match, then to manual selection.
